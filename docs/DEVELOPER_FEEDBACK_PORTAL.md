# Developer Feedback Portal Architecture & AI Guidelines

## Overview
The Developer Feedback Portal is an AI-first diagnostic tool built directly into the Sentinel1 application. Its primary purpose is to provide an external AI agent (like Google Jules) with high-fidelity, actionable data when a developer or tester reports a UI bug or requests a feature augmentation.

Instead of relying on human descriptions, the portal captures a complete "State Dump" of the application at the exact moment the feedback is triggered.

## The State Dump Structure
When the user taps the floating bug icon, the system generates a snapshot that is uploaded to Firestore (`dev_feedback` collection) and Firebase Storage (for screenshots). 

The state dump contains two primary artifacts:

### 1. Visual Ground Truth (Screenshot)
- A high-resolution PNG captured via a global `RepaintBoundary`.
- **Purpose**: Allows the AI to "see" what the user is seeing.
- **Location**: Uploaded to Firebase Storage. The download URL is included in the JSON metadata.

### 2. UI Index & Theme Metadata (JSON)
- Generated by the `UIIndexer` engine, this JSON payload breaks down the application into an AI-readable schema.
- **Purpose**: Provides exact structural, spatial, and styling data so the AI doesn't have to guess how elements are constructed or positioned.

#### JSON Schema Breakdown:
*   `screen_name`: The current active route/screen.
*   `device_info`: Logical resolution (width, height, pixel ratio).
*   `theme_context`: The Material Design 3 (M3) state.
    *   `typography`: All active text styles (font size, weight, letter spacing) for headlines, titles, body, etc.
    *   `colors`: The active semantic color tokens (primary, secondary, error, surface, etc.). **Agents MUST use these tokens instead of hardcoded colors to maintain UI consistency.**
*   `ui_hierarchy`: A flattened, recursive index of key widgets on the screen.
    *   `type`: The widget type (e.g., `Text`, `ElevatedButton`, `Container`).
    *   `bounds`: The exact spatial coordinates `(x, y, width, height)` of the widget relative to the screen.
    *   `text_content`: (If applicable) The readable text inside the widget.
    *   `widget_id`: A unique identifier for the widget instance.

## Guidelines for AI Agents (Jules)
When interpreting a ticket generated by this portal, adhere to the following workflow:

1. **Contextualize**: Load the provided JSON metadata and open the `screenshot_url` to visually confirm the state.
2. **Locate**: Use the `ui_hierarchy` to cross-reference the user's request (e.g., "Fix the alignment of the Submit button") with the exact `bounds` of that widget in the layout.
3. **Augment**: If the user asks for a new feature (e.g., "Add a search bar above the list"), find the relevant parent container in the JSON.
4. **Style**: **NEVER** invent colors or text styles. Extract the exact required styling from the `theme_context` object within the JSON to ensure your code matches the current Sentinel1 MD3 design system.

## Code Pointers
- **Trigger**: `lib/core/feedback/feedback_overlay.dart` (Contains the floating FAB and the RepaintBoundary wrapper).
- **Engine**: `lib/core/feedback/ui_indexer.dart` (Contains the recursive logic to traverse the widget tree and extract layout/theme data).
